Overview

The SSCC dataset contains synchronized audio and vibration recordings collected from a laboratory single-speed chain conveyor system under diverse operating conditions. Each clip corresponds to a condition defined by conveying speed, load level, operating category (normal or fault type), and noise setting. SSCC is designed to support both fault detection (normal vs abnormal) and multi-class fault classification, enabling research on multimodal representation learning and fusion under realistic domain shifts.

Data specification

  • Recording setup: The overall sensor placement and system layout are illustrated in Fig. 2. Audio is captured using three devices: Zoom H5 (WAV, 44.1 kHz), iPhone 11 (MP4, 44.1 kHz), and Xiaomi Mi 9 SE (MP4, 48 kHz). Vibration signals are acquired using one single-axis sensor and one tri-axis sensor (four channels total) sampled at 100 kHz (CSV format). Two loudspeakers are used to replay real factory noise recordings to simulate realistic environmental noise conditions. Only audio tracks are used from mobile recordings.
  • Segmentation: Data are collected under steady-state operating conditions and segmented into fixed-length clips (5 seconds per clip). No temporal overlap is applied between adjacent segments. All seven channels corresponding to the same time window are treated as synchronized modalities of a single sample.
  • Labels/attributes: normal + 4 fault types; operating attributes (speed, load, noise).
  • Splits: fault detection uses normal vs abnormal; fault classification uses separate multi-class splits.
  • File structure: ASDdataset/{normal|fault_type}/{condition}/{android|ios|audio|vibration}/

Benchmark

We provide two benchmark tracks with task-specific splits and evaluation protocols. Code is available at: this repository.

Fault detection (normal vs abnormal)

Train on normal clips only. Evaluate on a mixed test set containing both normal and abnormal clips.

# Fault detection (example) python -m pip install -r requirements.txt python run_detection.py --data ./dataset_root \ --train_split ./dataset_root/splits/detection_train.txt \ --test_split ./dataset_root/splits/detection_test.txt

Fault classification (multi-class)

Use a dedicated split for multi-class classification (normal + fault types, or fault-only depending on your setup).

# Fault classification (example) python -m pip install -r requirements.txt python run_classification.py --data ./dataset_root \ --train_split ./dataset_root/splits/classification_train.txt \ --test_split ./dataset_root/splits/classification_test.txt
